# 路线图与里程碑（Roadmap）

日期：2026-05-13

## 目标
在 10 周内交付可用于生产的 MVP，并在后续 4-6 周内完成关键增强功能（API、更多格式、计费）。

## 计划（建议）

- 准备阶段（Week 0，1 周）
  - 确认需求与 MVS（最小可行方案），选型（libvips、队列、对象存储），搭建最简 infra 原型。
  - 验收：基础 infra（S3/兼容、Redis、简单后端）可接收上传并返回任务 ID。

- Sprint 1（Weeks 1-2，2 周） — 单文件转换（MVP 核心）
  - 功能：拖拽上传、单文件转换、质量/尺寸设置、预览、单文件下载。
  - 验收：主流程端到端工作且界面友好。

- Sprint 2（Weeks 3-4，2 周） — 批量处理与队列
  - 功能：多文件队列、并行处理、ZIP 批量下载、进度显示。
  - 验收：批量任务稳定、ZIP 生成正确。

- Sprint 3（Weeks 5-6，2 周） — 用户账号与历史（可选）
  - 功能：注册登录、任务历史、重用配置、简单配额管理。
  - 验收：登录用户可查看历史并重新下载结果。

- Sprint 4（Weeks 7-8，2 周） — 性能优化与更多格式
  - 功能：引入 AVIF/HEIC 支持、优化并发与内存使用、CDN 加速方案。
  - 验收：引入一种新格式且整体性能显著提升。

- Sprint 5（Weeks 9-10，2 周） — API 与付费能力
  - 功能：REST API、Webhook、API Key 管理、Stripe 集成（基础订阅）。
  - 验收：外部系统可调用 API 提交任务并通过 webhook 接收完成通知。

- 交付与验证（Week 11，1 周）
  - 功能：全面验收测试、性能压力测试、安全扫描、上线准备。

## 优先级与风险
- 优先级（高→低）：核心上传/转换/下载 → 批量/队列 → API/计费 → 企业功能。
- 风险：大文件处理（内存/超时）、格式兼容性（HEIC/AVIF）、安全（恶意文件）。

## 资源建议
- 最小团队：1 PM、1-2 前端、1 后端、1 运维/QA。根据并发/业务规模扩充。
