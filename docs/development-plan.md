# 开发计划

本文档基于 [PRD.md](/C:/Users/29486/Desktop/photochanger/PRD.md) 拆解为可执行版本，并同步记录当前已完成阶段。

## 1. 目标与范围

当前目标是在仓库内逐步交付完整 MVP。阶段策略如下：

- 先打通核心主流程：上传 -> 设置 -> 转换 -> 下载
- 再补齐多文件队列、批量下载和前端工程化
- 最后接入异步队列、账户体系与开放 API

## 2. 技术决策

- 后端采用 FastAPI，便于快速定义 API 和任务元数据结构。
- 图片处理采用 Pillow，优先覆盖 JPEG、PNG、WebP、GIF 的主流场景。
- 前端已从原生静态页升级为 Vue 3 + Vite，便于承载队列、状态和后续复杂交互。
- 开发环境存储先使用本地文件系统；生产再切换对象存储。
- 当前任务状态依旧以内存为主；后续升级 Redis + 数据库。

## 3. 分阶段实施

### Phase 0：项目骨架与规范

状态：已完成

交付：
- 基础前后端目录
- 转换服务骨架
- 开发计划与启动说明

### Phase 1：单文件转换 MVP

状态：已完成

能力：
- 单文件上传
- 输出格式、质量、尺寸、EXIF 设置
- 原图与结果预览
- 单文件下载

接口：
- `POST /api/v1/convert`
- `GET /api/v1/tasks/{task_id}`
- `GET /api/v1/tasks/{task_id}/result`

### Phase 2：批量处理与队列

状态：本轮完成开发版

能力：
- 多文件队列
- 队列状态展示
- 批量转换后的 ZIP 打包下载
- 前端总体进度展示

接口：
- `POST /api/v1/convert`
- `POST /api/v1/batches`
- `GET /api/v1/batches/{batch_id}`
- `GET /api/v1/batches/{batch_id}/result`

当前实现说明：
- 前端负责队列编排与进度管理
- 后端负责结果归档与 ZIP 生成
- 这是 Sprint 2 的过渡方案，后续会替换为真正的异步 worker

### Phase 3：账号、历史与配置复用

目标：
- 用户登录
- 历史任务列表
- 重用转换参数
- 匿名/登录配额差异化

### Phase 4：格式扩展与性能优化

目标：
- 补齐 HEIC/AVIF
- 引入大文件异步处理
- 优化内存占用与并发
- 对接 CDN / 对象存储

### Phase 5：开放 API 与商业化能力

目标：
- API Key
- Webhook
- 订阅与配额
- 审计与删除策略

## 4. 当前实现策略

当前仓库已经完成：

- Phase 0：项目骨架
- Phase 1：单文件转换闭环
- Phase 2：Vue 工程化前端与批量 ZIP 下载开发版

下一步推荐优先顺序：

1. 接入 Redis 队列，替换前端主导的批处理模式
2. 落 PostgreSQL 保存任务与批次历史
3. 扩展 HEIC/AVIF 支持
4. 增加用户体系与历史页

## 5. 已知限制

- HEIC/AVIF 仍需额外编解码支持
- 批处理仍是请求内执行，不是真正的后台异步任务
- 任务和批次状态存储为内存级，重启服务会丢失
- 暂未接入数据库、对象存储和用户体系
