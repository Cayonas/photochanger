# 产品需求文档（PRD） — 在线图片格式转换工具

日期：2026-05-13
作者：产品经理

## 一、概述
在线图片格式转换工具（Web）旨在为个人用户、内容运营与开发者提供：
- 在浏览器中快速完成图片格式互转与压缩优化；
- 支持批量与异步任务、可编程 API 集成；
- 注重隐私与易用性，默认短期存储并提供数据删除能力。

核心价值：操作简洁、输出体积可控、支持自动化流水线、保护用户隐私。

## 二、目标用户与场景
- 普通用户：把手机拍摄的图片转为分享友好的格式并压缩。
- 内容/前端工程师：批量生成 Web 优化图片（WebP/AVIF、多分辨率）。
- 开发者/企业：通过 API 集成到媒体处理流水线中自动化转换。

## 三、成功指标
- 功能可用性：主流程（上传→设置→转换→下载）成功率 ≥ 99%。
- 性能：单文件（≤5MB）平均转换时延 < 3s（MVP 目标）。
- 可靠性：任务失败率 < 1%。
- 商业：注册转化率、付费转化率、每月活跃用户等。

## 四、MVP 功能清单与验收标准
1. 上传
   - 支持拖拽、文件选择、剪贴板粘贴、图片 URL 导入（受限）。
   - 验收：至少 Chrome/Edge/Firefox 上能完成拖拽并列出文件元信息。

2. 格式转换（MVP）
   - 支持 JPEG、PNG、HEIC、WebP、GIF（动画保留/静帧）。
   - 验收：转换后的文件能在本地打开，格式与用户选择一致。

3. 单/批量处理
   - 支持多文件队列、并行/序列处理、队列与总体进度显示；批量完成后可打包为 ZIP。
   - 验收：上传 10 张图片后能并行转换并生成 ZIP 下载。

4. 基本编辑选项
   - 输出格式选择、质量（压缩）滑块、尺寸调整（固定/按比/百分比）、裁剪（自由/比例）、保留/删除 EXIF。
   - 验收：对同一文件使用不同质量/尺寸能明显影响输出体积与分辨率。

5. 预览与比较
   - 转换前后缩略图、分辨率与体积对比。验收：界面展示原始/结果缩略与数值对比。

6. 错误与限额
   - 单文件失败不影响整体队列，明确错误提示；匿名用户单文件上限（示例 50MB）。
   - 验收：上传损坏文件时返回可读错误；超限文件被阻止并提示。

7. 基础安全
   - HTTPS、MIME 与文件头校验、输入大小限制、短期存储（默认 24 小时自动删除）。

## 五、高级功能（v1+）
- 支持更多格式（AVIF、TIFF、SVG）；
- 动画处理（GIF ↔ Animated WebP，帧率控制）；
- 多分辨率输出（自动生成 1x/2x/responsive）；
- REST API 与 Webhook 回调；
- 用户账户、API Key、计费订阅、任务历史；
- 智能压缩（目标体积、视觉感知优化）；
- 企业功能（SSO、私有化部署）。

## 六、非功能需求
- 可扩展性：转换 worker 可横向扩展（容器或 Serverless）；
- 性能：支持并发任务扩展；小文件秒级响应，批量异步处理；
- 可用性：目标 99.9% 可用性；
- 可访问性：响应式 UI，屏幕阅读器友好；
- 国际化：至少中文/英文；
- 合规：支持用户数据删除请求，满足 GDPR 基本要求。

## 七、核心交互流程（主流程）
1. 用户访问转换页面；
2. 上传（拖拽/选择/粘贴/URL）；
3. 在设置面板选择目标格式、质量、尺寸与元数据策略；
4. 点击“转换”，后台排队/执行；
5. 转换完成后显示可预览结果，提供单文件下载或批量 ZIP 下载；
6. 登录用户可在历史中查看并重用配置。

## 八、API 概览（示例）
- POST `/api/v1/convert` — multipart/form-data 上传文件或 JSON 提交 URL 与设置；返回 `task_id`。
- GET `/api/v1/tasks/{task_id}` — 查询任务状态与结果元数据。
- GET `/api/v1/tasks/{task_id}/result` — 获取结果下载链接（临时）。
- POST `/api/v1/webhooks` — 注册回调（已登录用户）。

身份验证：API Key（Bearer）或 OAuth/JWT（登录用户）。

示例：
```
POST /api/v1/convert
Headers: Authorization: Bearer <API_KEY>
Body: multipart form-data: file=@image.jpg; fields: target_format=webp; quality=80
```

-## 九、技术架构建议
- 前端：Vue（Vite/Nuxt） + 轻量组件库；采用 File API 处理本地预览；
- 后端：Node.js（Nest/Express）或 Python（FastAPI）；
- 图片处理：`libvips`（Node: `sharp` / Python: `pyvips`）；
- 异步队列：Redis + Bull/BullMQ 或 SQS + Lambda；
- 存储：S3 或兼容对象存储 + CDN；
- 数据库：PostgreSQL 存任务与用户元数据；
- 监控：Prometheus/Grafana，错误追踪 Sentry；
- 部署：容器化（Docker + K8s）或 Serverless（按需）。

## 十、安全与隐私
- 强制 HTTPS；传输与静态存储加密；
- 白名单格式与文件头校验，限制可执行内容；
- 默认删除策略：上传文件默认 24 小时后自动删除（可配置）；
- 提供用户数据删除 API 与隐私说明，记录审计日志。

## 十一、监控与度量
- 转换次数、成功率、失败率、平均延时、队列长度、存储使用量；
- 业务：日活、付费转化、每用户平均消耗。

## 十二、测试与验收
- 功能测试：常见格式互转、批量边界（超大、损坏）、元数据处理；
- 性能测试：并发小文件压力测试、单大文件性能与内存使用；
- 安全测试：输入校验、依赖漏洞扫描、渗透测试；
- UAT：主流程在主流浏览器与移动端可用。
